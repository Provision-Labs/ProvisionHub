# provision-cli

ProvisionHub Plugin Manager CLI - Manage plugins with a single command.

## Features

- **Generate** `plugins.json` from plugin `conf.json` manifests automatically
- **Install** plugins from `registry.provisionlabs.dev` (coming soon)
- **Detect** plugin directories recursively
- **Validate** plugin configurations
- **Portable** - single binary, zero dependencies outside stdlib

## Installation

### From Source

```bash
cd tools/provision-cli
go build -o ~/bin/provision-cli .
```

### Or use via make

```bash
make build-provision-cli
# Binary at: ./bin/provision-cli
```

## Usage

### Generate plugins.json

Generate `plugins.json` from all plugin `conf.json` manifests:

```bash
# From project root (auto-detects paths)
provision-cli generate

# With custom paths
provision-cli generate \
  --plugins-dir ./plugins \
  --output ./plugins.json

# With specific project root
provision-cli generate --root /path/to/ProvisionHub
```

**What it does:**

- Scans `plugins/` directory recursively
- Reads each plugin's `conf.json`
- Generates standardized `plugins.json` with:
   - Correct working directories
   - Transport endpoints
   - Metadata and versioning
   - Environment variables for plugin startup

## Plugin conf.json Format

Each plugin must have a `conf.json` manifest:

```json
{
   "id": "keycloak-go",
   "creator": "Your Name",
   "contact": "your@email.com",
   "type": "auth",
   "language": "go",
   "title": "Keycloak IdP for ProvisionHub",
   "description": "Keycloak plugin for authentication",
   "loadMode": "persistent",
   "runtime": {
      "start": {
         "command": "go",
         "args": ["run", "./cmd/plugin"]
      },
      "transport": {
         "protocol": "grpc",
         "address": "127.0.0.1:50051",
         "insecure": true,
         "timeoutMs": 2500
      }
   },
   "behavior": {
      "restartPolicy": "on-failure",
      "maxRestarts": 5,
      "healthProbe": {
         "rpc": "Health",
         "intervalMs": 10000,
         "timeoutMs": 1500
      }
   },
   "version": "0.1.0",
   "repository": {
      "type": "git",
      "url": "https://github.com/provisionlabs/official-plugins/keycloak-go"
   }
}
```

### Field Reference

| Field                         | Type   | Required | Description                                       |
| ----------------------------- | ------ | -------- | ------------------------------------------------- |
| `id`                          | string | Yes      | Unique plugin identifier (lowercase, kebab-case)  |
| `type`                        | string | Yes      | Plugin type (auth, provisioning, etc.)            |
| `language`                    | string | Yes      | Implementation language (go, python, etc.)        |
| `title`                       | string | Yes      | Human-readable plugin title                       |
| `description`                 | string | Yes      | Short description of what plugin does             |
| `loadMode`                    | string | Yes      | `persistent` (auto-start) or `on-demand`          |
| `version`                     | string | Yes      | Semantic version (e.g., 0.1.0)                    |
| `creator`                     | string | Yes      | Plugin author name                                |
| `contact`                     | string | Yes      | Author email or contact                           |
| `runtime.start.command`       | string | Yes      | Command to execute (`go`, `python`, `node`, etc.) |
| `runtime.start.args`          | array  | Yes      | Arguments to pass (`["run", "./cmd/plugin"]`)     |
| `runtime.transport.protocol`  | string | Yes      | Transport protocol (`grpc`, `http`, etc.)         |
| `runtime.transport.address`   | string | Yes      | Listen address (`127.0.0.1:50051`)                |
| `runtime.transport.insecure`  | bool   | No       | Allow insecure connections (default: false)       |
| `runtime.transport.timeoutMs` | number | No       | RPC timeout in milliseconds                       |
| `behavior.restartPolicy`      | string | No       | `on-failure`, `always`, `never`                   |
| `behavior.maxRestarts`        | number | No       | Max restart attempts before giving up             |
| `behavior.healthProbe.rpc`    | string | No       | Health check RPC method name                      |
| `repository`                  | object | No       | Plugin repository metadata                        |

## Integration with Makefile

Add to your Makefile:

```makefile
# Build provision-cli
build-provision-cli:
	@mkdir -p bin
	@cd tools/provision-cli && go build -o ../../bin/provision-cli .
	@echo "вњ“ provision-cli built at bin/provision-cli"

# Generate plugins.json (runs before dev)
generate-plugins:
	@./bin/provision-cli generate || go run ./tools/provision-cli generate

# Development setup
dev-setup: build-provision-cli generate-plugins dev-infra-up
```

## Integration with CI/CD

### GitHub Actions Example

```yaml
- name: Generate plugins.json
  run: |
     go run ./tools/provision-cli generate
     git diff --exit-code apps/control-plane/plugins.json || \
       (echo "plugins.json is out of sync" && exit 1)
```

### Adding new plugins

1. Create plugin directory: `plugins/{type}/{id}/`
2. Add `conf.json` with plugin metadata
3. Run `provision-cli generate` (or `make generate-plugins`)
4. Commit updated `plugins.json`
